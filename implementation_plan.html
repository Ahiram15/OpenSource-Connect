📄 OpenSource Connect - Implementation Plan

This plan is written in simple English. It details how you and your partner can use the Antigravity AI Coding Assistant and Git / GitHub to build the project together from scratch.

🛠️ Our Technology Stack

  • Frontend (Client): Built using React.js (Vite) and styled with Tailwind CSS. Custom language graphs will use Recharts.
  • Backend (Server): Running on Node.js and Express.js.
  • Database: MongoDB (used to save profile data and saved issues).
  • APIs: GitHub API (to load open issues) and Google Gemini API (to generate recommendations and learning roadmaps).

🤖 How to use Antigravity (AI Assistant)

Since both of you have the Antigravity assistant in your editor, you can ask it to write code for you. Use these simple prompts:

  1. Tell it the exact file name (e.g. Create client/src/pages/Dashboard.jsx).
  2. Explain that you are using React and Tailwind CSS.
  3. Describe the design (e.g. "Use a dark theme with a glass-like header card and purple buttons").

🤝 How to Use Git & GitHub (Step-by-Step)

Think of GitHub like a shared Google Drive for your code files. Follow these steps to collaborate:

1 Install Tools & Configure Identity

Download and install Git from git-scm.com. Then, open your terminal inside VS Code and run these config commands:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
2 Copy the Repository (Clone)

Both partners run this command to copy the shared files from GitHub to their computers:

git clone https://github.com/Ahiram15/OpenSource-Connect.git
3 Create Your Private Branch

Never write code directly on the main branch. Always work on your own branch to avoid conflicts:

Partner 1 runs:

git checkout -b feature/frontend-user-space

Partner 2 runs:

git checkout -b feature/frontend-issues
4 Save & Upload Your Daily Code

Run these three commands to save and upload your changes at the end of each session:

git add .
git commit -m "feat: Add styles and cards to dashboard"
git push origin feature/your-branch-name
5 Pull Changes from Your Partner

To load code updates that your partner has merged into main, run:

git checkout main
git pull origin main
git checkout feature/your-branch-name
git merge main

🎨 Phase 1: Frontend Development UI (Both Partners)

We work on the user interface screens first. To avoid merge conflicts, we divide the screens page-by-page:

👤 Partner 1: User Layouts
  • Login Screen (Login.jsx): Github OAuth sign-in page.
  • Dashboard (Dashboard.jsx): Panels displaying user skills and language charts.
  • Profile (Profile.jsx): User interest options checklist.
🔍 Partner 2: Issue Lists
  • Issues Feed (IssueList.jsx): Listing of open issues with Match Score circular progress loops.
  • Filters: Sorting sidebar by language and difficulty tags.
  • Roadmap Details (IssueDetail.jsx): Full issue text details alongside learning checkpoints.

⚙️ Phase 2: Backend Development API (Both Partners)

Once all frontend page layouts are merged into main, move to the server/ folder to build APIs:

🛡️ Partner 1: Servers & Auth
  • Setup express server entry points in index.js.
  • Create Mongoose database schemas (Users & Bookmarks).
  • Create GitHub OAuth token authorization endpoints.
🧠 Partner 2: GitHub API & AI
  • Configure GitHub API search service triggers.
  • Build Gemini AI prompt filters to generate roadmaps.
  • Create REST endpoints like /api/recommendations.

📈 Tips to Stay Consistent

  1. Design System Colors: Save color variables like --bg-dark: #0b0f19, --primary: #6366f1 (indigo accent), and --success: #10b981 (match green) in client/src/index.css.
  2. Track Tasks: Use task.md checklist items daily (use [/] for in-progress, [x] for complete).
  3. 5-Minute Daily Syncs: Discuss what was built, what's next, and if any files are conflicting.
